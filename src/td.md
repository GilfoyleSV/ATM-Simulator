# Техническое задание: Симулятор банкомата (ATM Simulator)

## 1. Цель проекта

Разработать консольное Java-приложение, имитирующее работу банковского автомата, для практики объектно-ориентированного проектирования, работы со строгой типизацией, коллекциями и точными финансовыми вычислениями.

---

## 2. Функциональные требования

### 2.1. Аутентификация пользователя

* При запуске приложение требует ввести номер счета (id) и PIN-код.
* Система должна сверять введенные данные со списком существующих аккаунтов.
* При совпадении данных открывается доступ к главному меню. При несовпадении — выводится ошибка, и приложение возвращается к запросу авторизации.

### 2.2. Главное меню (Операции)

После успешного входа пользователю доступны следующие операции:

1. **Проверить баланс:** Вывод текущего остатка средств на счете.
2. **Пополнить счет:** Ввод суммы с консоли и увеличение баланса аккаунта.
3. **Снять наличные:** Ввод суммы с консоли и уменьшение баланса аккаунта (с проверкой на наличие достаточного количества средств).
4. **Перевод средств:** Ввод номера счета получателя и суммы. Списание средств у текущего пользователя и зачисление на счет получателя (с проверкой существования получателя и достаточности средств).
5. **История транзакций:** Вывод списка всех совершенных операций по текущему счету.
6. **Выход:** Возврат к экрану аутентификации.

### 2.3. Ограничения на финансовые операции

* Сумма пополнения, снятия и перевода должна быть строго больше нуля.
* Баланс аккаунта не может становиться отрицательным.

---

## 3. Требования к архитектуре и коду (Java-специфика)

### 3.1. Типы данных

* **Денежные средства:** Все балансы и суммы транзакций должны храниться и обрабатываться исключительно с использованием класса `java.math.BigDecimal`. Использование `float` или `double` запрещено.

### 3.2. Структура классов (ООП)

Приложение должно быть разделено как минимум на следующие сущности:

1. **`Account` (Класс-сущность)**
* Поля: `String id` (или `accountNumber`), `String pin`, `BigDecimal balance`, `List<Transaction> transactionHistory`.
* Методы: Геттеры, методы для изменения баланса (дебет/кредит), метод добавления транзакции в историю.


2. **`Transaction` (Класс-сущность / Запись)**
* Поля: `String type` (ПОПОЛНЕНИЕ, СНЯТИЕ, ПЕРЕВОД_ИСХОДЯЩИЙ, ПЕРЕВОД_ВХОДЯЩИЙ), `BigDecimal amount`, `java.time.LocalDateTime timestamp`, `String details` (например, "Перевод на счет X").


3. **`Bank` (Класс бизнес-логики)**
* Поля: `Map<String, Account> accounts` (хранилище всех аккаунтов в памяти, где ключ — номер счета).
* Методы: `authenticate(String id, String pin)`, `getAccount(String id)`, метод инициализации (наполнение карты `accounts` 2-3 тестовыми аккаунтами при старте).


4. **`ATMService` (Класс управления банкоматом)**
* Содержит логику выполнения операций: проверку лимитов, вызов методов `Bank` и `Account`, логирование транзакций.


5. **`Main` (Точка входа)**
* Инициализация `Bank` и `ATMService`.
* Запуск бесконечного цикла консольного меню (`java.util.Scanner`) для взаимодействия с пользователем.



### 3.3. Коллекции

* Для хранения аккаунтов в классе `Bank` использовать `java.util.HashMap` (обеспечивает поиск аккаунта за $O(1)$).
* Для хранения истории транзакций в классе `Account` использовать `java.util.ArrayList`.

---

## 4. Обработка ошибок и исключений

Приложение не должно аварийно завершаться (падать) при некорректном вводе. Необходимо обрабатывать:

* Ввод букв вместо чисел при запросе сумм или пунктов меню (`InputMismatchException`).
* Попытку снять сумму, превышающую текущий баланс.
* Попытку перевода на несуществующий номер счета.
* Ввод отрицательных или нулевых сумм.
